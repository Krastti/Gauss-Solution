# Algebra1: сравнение методов решения СЛАУ

Проект реализует лабораторную работу по сравнению методов решения систем линейных алгебраических уравнений:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с частичным выбором главного элемента по столбцу;
- LU-разложение без перестановок;
- решение треугольных систем прямой и обратной подстановкой.

Код написан на C11, рассчитан на Windows и собирается через CMake.

## Структура проекта

- `src/gauss` — метод Гаусса, прямой и обратный ход, подстановки, невязка `||Ax - b||`.
- `src/lu` — LU-разложение, решение через `Ly = b` и `Ux = y`.
- `src/matgen` — генерация случайных матриц, случайных векторов, матриц Гильберта, правой части по точному решению.
- `src/timer` — высокоточный таймер Windows на базе `QueryPerformanceCounter`.
- `src/logging` — простой логгер.
- `test` — юнит-тесты публичных функций.
- `data/reports` — отчёты и CSV-файлы с результатами экспериментов.

## Сборка

```bash
cmake -S . -B build
cmake --build build
```

По умолчанию вместе с программой собирается исполняемый файл тестов `Algebra1Tests`.

Чтобы отключить сборку тестов:

```bash
cmake -S . -B build -DALGEBRA1_BUILD_TESTS=OFF
cmake --build build
```

## Запуск экспериментов

```bash
build\Debug\Algebra1.exe
```

Для одноконфигурационных генераторов путь обычно короче:

```bash
build\Algebra1.exe
```

Программа сохраняет результаты в CSV-файлы:

- `data/reports/single_system_experiment.csv` — сравнение времени решения одной системы для разных `n`;
- `data/reports/multiple_rhs_experiment.csv` — сравнение метода Гаусса с LU при нескольких правых частях;
- `data/reports/hilbert_accuracy_experiment.csv` — проверка точности на матрицах Гильберта.

## Запуск тестов

```bash
ctest --test-dir build
```

Или напрямую:

```bash
build\Debug\Algebra1Tests.exe
```

## Формат результатов

Для одной системы CSV содержит:

- `n` — размер системы;
- `method` — метод решения;
- `status` — код результата;
- `total_ms` — общее время;
- `decompose_ms` — время LU-разложения, если применимо;
- `solve_ms` — время решения после подготовки;
- `residual` — норма невязки.

Для нескольких правых частей CSV содержит:

- `n` — размер матрицы;
- `k` — число правых частей;
- `method` — метод решения;
- `status` — код результата;
- `total_ms` — общее время;
- `decompose_ms` — время однократного LU-разложения;
- `solve_ms` — суммарное время подстановок или решений.

Для матриц Гильберта CSV содержит:

- `n` — размер матрицы;
- `method` — метод решения;
- `status` — код результата;
- `relative_error` — относительная погрешность `||x_approx - x|| / ||x||`;
- `residual` — норма невязки `||Ax - b||`.

## Примечания

LU-разложение реализовано без перестановок, как указано в задании. Поэтому для матриц с нулевым или слишком малым ведущим элементом LU может вернуть `LU_SINGULAR`, даже если метод Гаусса с выбором ведущего элемента способен решить систему.

Случайные матрицы и векторы генерируются детерминированным локальным генератором по фиксированному `seed`, без изменения глобального состояния `rand`.
